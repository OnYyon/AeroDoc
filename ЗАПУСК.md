# Инструкция по запуску AeroDoc Assistant

## Вариант 1: Автоматический запуск (Windows)

### Шаг 1: Запустите Backend
Дважды кликните на файл `start_backend.bat` или выполните в терминале:
```cmd
start_backend.bat
```

Backend будет доступен на: **http://localhost:8000**

### Шаг 2: Запустите Frontend (в новом окне терминала)
Дважды кликните на файл `start_frontend.bat` или выполните в терминале:
```cmd
start_frontend.bat
```

Frontend будет доступен на: **http://localhost:3000**

---

## Вариант 2: Ручной запуск

### Backend (Терминал 1)

```bash
# Перейдите в директорию backend
cd backend

# Создайте виртуальное окружение (если еще не создано)
python -m venv venv

# Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
# source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt

# Запустите сервер
python main.py
```

Backend будет доступен на: **http://localhost:8000**

### Frontend (Терминал 2 - новый терминал!)

```bash
# Перейдите в директорию frontend
cd frontend

# Установите зависимости (если еще не установлены)
npm install

# Запустите dev-сервер
npm run dev
```

Frontend будет доступен на: **http://localhost:3000**

---

## После запуска

1. Откройте браузер и перейдите на **http://localhost:3000**
2. Зарегистрируйтесь, выбрав роль (engineer, technician, quality_control, project_manager, maintenance, admin)
3. Загрузите документы через кнопку "Загрузить документ"
4. Создайте новый чат и начните общение

---

## Требования

- **Python 3.8+** (проверьте: `python --version`)
- **Node.js 16+** (проверьте: `node --version`)
- **npm** (обычно устанавливается вместе с Node.js)

---

## Устранение проблем

### Backend не запускается
- Убедитесь, что Python установлен: `python --version`
- Убедитесь, что порт 8000 свободен
- Проверьте, что все зависимости установлены: `pip install -r requirements.txt`

### Frontend не запускается
- Убедитесь, что Node.js установлен: `node --version`
- Убедитесь, что порт 3000 свободен
- Удалите `node_modules` и `package-lock.json`, затем выполните `npm install` заново

### Ошибки при отправке сообщений
- Это нормально, если ML-сервис не запущен
- Система покажет сообщение об ошибке, но остальной функционал будет работать
- Для работы с ML-сервисом настройте `ML_SERVICE_URL` в файле `.env` в директории `backend`

---

## Примечание о ML-сервисе

По умолчанию система ожидает ML-сервис на `http://localhost:8001/api/ml/process`.

Если ML-сервис не запущен, при отправке сообщений будет показано сообщение об ошибке, но загрузка документов и просмотр истории чатов будут работать нормально.
