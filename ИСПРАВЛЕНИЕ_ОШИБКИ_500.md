# Исправление ошибки 500 при регистрации

## Быстрое решение

### Шаг 1: Остановите Backend сервер
Нажмите `Ctrl+C` в терминале, где запущен backend

### Шаг 2: Удалите старую базу данных (если есть)
```bash
cd backend
del aerodoc.db
```
Или вручную удалите файл `backend/aerodoc.db`

### Шаг 3: Инициализируйте базу данных
```bash
python init_db.py
```

Должно вывести:
```
Создание таблиц в базе данных...
✓ Таблицы успешно созданы!
✓ База данных готова к использованию
```

### Шаг 4: Запустите Backend заново
```bash
python main.py
```

Или используйте:
```bash
start_backend.bat
```

### Шаг 5: Попробуйте зарегистрироваться снова

---

## Альтернативный способ (если первый не помог)

### Проверьте, что все зависимости установлены:

```bash
cd backend
pip install -r requirements.txt
```

Убедитесь, что установлены:
- fastapi
- sqlalchemy
- pydantic
- pydantic-settings
- python-jose
- bcrypt
- python-multipart
- httpx

### Проверьте версию Python:

```bash
python --version
```

Должна быть версия 3.8 или выше.

---

## Проверка состояния сервера

После запуска backend, откройте в браузере:

**http://localhost:8000/api/health**

Должен вернуться JSON:
```json
{
  "status": "ok",
  "database": "connected",
  "tables_created": true
}
```

Если видите ошибку, значит проблема с базой данных.

---

## Если проблема сохраняется

1. **Проверьте логи backend** - в терминале должны быть видны ошибки
2. **Проверьте консоль браузера** (F12 → Console) - там может быть дополнительная информация
3. **Проверьте вкладку Network** (F12 → Network) - посмотрите ответ сервера на запрос регистрации

Скопируйте текст ошибки и сообщите разработчику.
