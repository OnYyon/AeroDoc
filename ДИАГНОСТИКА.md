# Диагностика проблемы регистрации

## Шаги для выявления проблемы

### 1. Проверьте, запущен ли Backend

Откройте браузер и перейдите на: **http://localhost:8000/docs**

Должна открыться страница Swagger UI с документацией API. Если страница не открывается, значит backend не запущен.

### 2. Проверьте консоль браузера

1. Откройте браузер (Chrome/Firefox)
2. Нажмите F12 для открытия DevTools
3. Перейдите на вкладку "Console"
4. Попробуйте зарегистрироваться снова
5. Посмотрите, какие ошибки выводятся в консоли

### 3. Проверьте вкладку Network

1. В DevTools перейдите на вкладку "Network"
2. Попробуйте зарегистрироваться
3. Найдите запрос к `/api/auth/register`
4. Кликните на него и посмотрите:
   - Status Code (должен быть 200 или ошибка)
   - Response (ответ сервера)
   - Request (что отправляется)

### 4. Проверьте логи Backend

В окне терминала, где запущен backend, должны быть видны:
- Запросы к API
- Ошибки (если есть)

### 5. Проверьте базу данных

Запустите скрипт проверки базы данных:

```bash
cd backend
python test_db.py
```

Должно вывести:
```
✓ База данных успешно создана/проверена
✓ Таблицы созданы успешно
✓ Подключение к базе данных работает
```

## Частые проблемы и решения

### Проблема: "Не удалось подключиться к серверу"

**Решение:**
- Убедитесь, что backend запущен на порту 8000
- Проверьте, что в терминале backend нет ошибок
- Убедитесь, что порт 8000 не занят другим приложением

### Проблема: "Пользователь с таким email уже существует"

**Решение:**
- Используйте другой email для регистрации
- Или удалите файл `backend/aerodoc.db` и перезапустите backend

### Проблема: Ошибка валидации данных

**Решение:**
- Убедитесь, что все поля заполнены
- Email должен быть валидным (например, test@example.com)
- Пароль должен быть минимум 6 символов
- Роль должна быть выбрана из списка

### Проблема: Ошибка базы данных

**Решение:**
1. Удалите файл `backend/aerodoc.db` (если существует)
2. Перезапустите backend
3. База данных создастся автоматически

## Что делать дальше

После выполнения диагностики:

1. **Скопируйте текст ошибки** из консоли браузера или терминала backend
2. **Сделайте скриншот** ошибки (если возможно)
3. **Опишите**, что именно вы делали перед появлением ошибки

Это поможет быстрее найти и исправить проблему.
